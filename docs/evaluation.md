# 평가 방법론

## 개요

librAIan의 검색 및 추천 성능을 체계적으로 평가하기 위한 방법론을 설명합니다. 평가에는 정량적 메트릭과 정성적 평가를 모두 포함합니다.

## 정량적 평가

### 메트릭 정의

#### Hit@K (Precision at K)
상위 K개 결과 중 관련 문서가 하나라도 있는지 평가합니다.

```
Hit@K = 1 if any(retrieved[:K] in relevant) else 0
```

**장점**: 직관적 이해 용이
**단점**: 순위 정보 무시

#### NDCG@K (Normalized Discounted Cumulative Gain)
순위별 관련성을 고려한 메트릭입니다.

```
DCG@K = sum(rel_i / log2(i+1) for i in 1 to K)
NDCG@K = DCG@K / IDCG@K
```

**IDCG**: 이상적인 순위에서의 DCG
**장점**: 순위 가중치 반영
**단점**: 계산 복잡

#### MRR (Mean Reciprocal Rank)
첫 번째 관련 문서의 역순위를 평균합니다.

```
MRR = (1/|Q|) * sum(1/rank_i for i in Q)
```

**장점**: 첫 번째 관련 문서 중요성 강조
**단점**: 하나의 문서만 고려

### 평가 데이터 구성

#### 쿼리 다양성
- **모호한 질의**: "힐링 에세이" (페르소나 의존)
- **구체적 질의**: "파이썬 입문서" (직접적)
- **복합 질의**: "고등학생 과학 소설" (여러 제약)

#### 페르소나 기반
- 직장인, 학생, 주부 등 다양한 사용자 그룹
- 시나리오별 평가: "퇴근 후", "주말", "여행 중"

#### 난이도 분류
- **Easy**: 명확한 관련 도서 존재
- **Medium**: 여러 후보 존재
- **Hard**: 주관적 판단 필요

## 정성적 평가

### LLM 심사관
GPT-4 등의 LLM을 사용하여 검색 결과의 품질을 평가합니다.

**평가 기준**:
- 관련성: 쿼리에 대한 적합성
- 다양성: 결과의 다양성
- 유용성: 실제 사용 가능성

### 사용자 설문
실제 사용자 대상으로 추천 만족도를 조사합니다.

**설문 항목**:
- 추천 정확도 (1-5점)
- 결과 다양성
- 인터페이스 사용성
- 전반적 만족도

## 실험 설계

### 비교 대상
1. **Baseline**: Dense 검색만 사용
2. **BM25**: 전통적 키워드 검색
3. **Hybrid**: Dense + BM25 결합
4. **Cross-Encoder**: 리랭킹 추가
5. **HCX**: LLM 리랭킹
6. **Full Pipeline**: 모든 컴포넌트 통합

### Ablation Study
각 컴포넌트의 기여도를 분석합니다.

**연구 질문**:
- 쿼리 이해 모듈이 성능을 얼마나 향상시키는가?
- 리랭킹이 검색 품질에 미치는 영향은?
- 도서관 API 연동의 실용성 향상도는?

### 통계적 검증
- t-test: 두 방법 간 유의미한 차이 검증
- ANOVA: 다중 그룹 비교
- 부트스트랩: 신뢰 구간 추정

## 평가 자동화

### 스크립트 구조
```python
# 평가 실행
evaluator = Evaluator(eval_data)
results = evaluator.evaluate_all_methods()

# 통계 분석
stats = StatisticalAnalyzer(results)
stats.print_summary()
stats.plot_comparison()
```

### CI/CD 통합
- PR마다 자동 평가 실행
- 성능 회귀 감지
- 결과 리포트 생성

## 결과 해석

### 성능 지표
- **목표**: NDCG@10 > 0.75, Hit@10 > 0.8
- **벤치마크**: 기존 시스템 대비 20% 향상

### 오류 분석
- False Positive: 관련 없는 결과 상위 노출
- False Negative: 관련 결과 누락
- 순위 오류: 관련성은 높으나 순위 낮음

- 데이터 추가 수집
