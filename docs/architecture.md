# 시스템 아키텍처

## 개요

librAIan은 RAG(Retrieval-Augmented Generation) 기반의 도서 추천 시스템으로, 다음과 같은 주요 컴포넌트로 구성됩니다:

1. **쿼리 이해 모듈**: 사용자 쿼리를 분석하고 의도를 파악
2. **검색 모듈**: 벡터 데이터베이스에서 관련 도서 검색
3. **리랭킹 모듈**: 검색 결과를 재정렬하여 품질 향상
4. **설명 생성 모듈**: 추천 이유를 자연어로 설명
5. **도서관 API 연동**: 실시간 대출 가능성 확인

## 데이터 흐름

1. **입력**: 사용자 쿼리 (텍스트)
2. **쿼리 이해**: 의도 분류, 페르소나 식별, 쿼리 확장
3. **검색**: 
   - Dense: 의미론적 유사도 기반 검색
   - BM25: 키워드 기반 검색
   - Hybrid: 두 방식의 결합
4. **리랭킹**: 
   - Cross-Encoder: 쿼리-문서 쌍 재평가
   - HCX: 대형 언어 모델 기반 리랭킹
   - Availability: 대출 가능성 우선순위
5. **출력**: 추천 도서 리스트 + 설명

## 기술 컴포넌트

### 벡터 데이터베이스 (Qdrant)
- 고성능 벡터 검색
- 실시간 인덱싱
- 분산 처리 지원

### 임베딩 모델 (Sentence Transformers)
- 다국어 지원 (한국어 포함)
- 도메인 특화 파인튜닝 가능
- 효율적인 벡터 변환

### 리랭킹 모델
- **Cross-Encoder**: 정확한 쌍별 점수 계산
- **HCX**: 맥락 이해 기반 리랭킹
- **LLM Listwise**: 전체 리스트 최적화

### 외부 API 연동
- **도서관정보나루**: 실시간 대출 정보
- **알라딘/예스24/네이버**: 도서 메타데이터 수집

## 확장성 고려사항

### 수평 확장
- 검색 모듈: Qdrant 클러스터링
- API 서버: FastAPI 인스턴스 복제
- 캐싱: Redis 도입

### 수직 확장
- 더 큰 모델: GPT-4, Claude 등
- 추가 데이터 소스: 해외 도서관 API
- 멀티모달: 이미지/오디오 기반 검색

## 보안 고려사항

- API 키 관리: 환경 변수 + 시크릿 매니저
- 데이터 암호화: 민감 정보 암호화 저장
- 접근 제어: JWT 토큰 기반 인증
- 로깅: 감사 로그 기록
